# Milestone v1.0: MVP

**Status:** SHIPPED 2026-03-05
**Phases:** 1-8 (+ 7.1 inserted)
**Total Plans:** 19

## Overview

Transform the @chillwhales/* monorepo from working-but-fragile code into publish-ready npm packages with professional infrastructure. The roadmap follows a strict dependency chain: fix builds first, establish code quality tooling, add structural validation, wire up coverage, automate CI, prepare package metadata, automate releases, and extract utilities from external repos. After Phase 7, packages are published on npm. Phase 8 consolidates code from consumer repos back into the monorepo.

## Milestone Summary

**Decimal Phases:**
- Phase 7.1: Rename LSP30→LSP31, LICENSE, Scripts (inserted after Phase 7 for package identity fix before extraction)

**Key Decisions:**
- Config package uses direct .ts exports (no build step) — unbuild/vitest consume TS natively
- Single default pnpm catalog (no named catalogs) — all shared deps meet 2+ package threshold
- Biome v2 with all formatting defaults (tabs, 80 width, double quotes, trailing commas)
- simple-git-hooks over husky — simpler config, package.json-based
- ESM-only build output — removed rollup.emitCJS, attw --profile esm-only
- @changesets/changelog-github for PR-attributed changelogs
- pkg-pr-new with fork guard for PR snapshot releases
- LSP17 prefix inlined as constant instead of adding @lukso dependency
- ERC725Y data-key utilities created fresh following LSP2 spec

**Issues Resolved:**
- 10 pre-existing lsp29 test failures (fixture alignment)
- publint/attw CJSResolvesToESM errors (removed main field, ESM-only)
- knip false positives (workspace-aware config, entry point detection)

**Issues Deferred:**
- 5 newer packages (lsp7, lsp8, lsp17, lsp26, up) missing READMEs
- LSP26 package candidate (4 marketplace functions)
- lsp-indexer metadata handler migration to @chillwhales parsers
- Cross-package integration tests

**Technical Debt Incurred:**
- Duplicated constants across lsp2, lsp29, lsp31
- parseInt for hex in lsp6 (precision loss >53 bits)
- Silent error swallowing in lsp6 parsers
- ERC725.js instantiation in lsp23 per call (should be module-level)
